Trigger Events

Send Integrated Urgent Care Report (POCD_TE000003GB01)

The Integrated Urgent Care Send trigger event indicates that a NHS Integrated Urgent Care CDA document is ready to send from the originating system.

Integrated Urgent Care Nullify By Replacement (POCD_TE000004GB01)

For the Integrated Urgent Care CDA Document to be nullified the trigger event is the realisation that the Integrated Urgent Care CDA Document was sent in error, in that it contained an error and should never have been sent and that no correct Integrated Urgent Care CDA Document is available to send.

Send Integrated Urgent Care Ambulance Request (REPC_TE000001GB01)

The Integrated Urgent Care Ambulance Request trigger event indicates that an ambulance request or an update to an existing ambulance request is sent.

Integrated Urgent Care Repeat Caller Query (QUPA_TE000001GB01)

The Integrated Urgent Care Repeat Caller Query trigger event indicates that a query of the repeat caller system is triggered.

Integrated Urgent Care Repeat Caller Response (PRPA_TE000001GB01)

The Integrated Urgent Care Repeat Caller Response trigger event indicates that a response from the repeat caller system is triggered.

Repeat Caller Identified (PRPA_TE000012GB01)

The Integrated Urgent Care Repeat Caller Identified trigger event indicates that a caller has been identified as a repeat caller.

Primary General Practitioner Recipient Integrated Urgent Care CDA Document (urn:nhs-itk:interaction:primaryGeneralPractitionerRecipientIntegratedUrgentCareCDADocument-v3-0)  

Communicates Integrated Urgent Care Report to a General Practitioner recipient where the recipient is recommended to act upon information carried in the document.

Note: This interaction has a message format that is constrained using HSCIC templates. This format is validated using a two level process. There are two sets of schema one for each level. The sender of this interaction must build all instances to the "on the wire" format using the generic CDA definition indicated in the wire format below. These instances must also conform to the rules stated in the conformance definition indicated in the conformance format below. The second level of validation requires transformation before schema validation can take place. For full details of this validation process please see the "Technical Guidance for Templated CDA Domains" section under the associated specifications tab.

Communicates Integrated Urgent Care repeat caller document set which includes the Integrated Urgent Care Reports from repeat caller system along with the current Integrated Urgent Care Report to an Out of Hour recipient where the recipient is recommended to act upon information carried in the document set. Distribution envelope will carry multipleIntegrated Urgent Care Reports as different payloads within the same envelope..

Note: Although the main message is not a CDA document, the imbedded CDA documents carried in this interaction have a message format that is constrained using HSCIC templates. This format is validated using a two level process. There are two sets of schema one for each level. The sender of this interaction must build all instances to the "on the wire" format using the generic CDA definition indicated in the wire format below. These instances must also conform to the rules stated in the conformance definition indicated in the conformance format below. The second level of validation requires transformation before schema validation can take place. For full details of this validation process please see the "Technical Guidance for Templated CDA Domains" section under the associated specifications tab.
