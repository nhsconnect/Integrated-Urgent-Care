Change History

Version: 2.2; Status: Final

  • Specification
    • The Pack has been rebranded to HSCIC.
  • Examples
    • Examples have been updated with more guidance.
  • Model
    • The fixed code of 285091000000108 carried in ClinicalDocument.code for nullification document type is now enforced in the model and schema.
    • The ParentDocument.code has been changed to CWE to allow for the document being nullified to use coding other systems as well as SNOMED CT.
    • As both the above changes are backward compatible the model had not been up versioned.
    • The Recipient Constraint Choice (NPFIT-800008#Role) has been upversioned for later domains, since templates, RecipientOrganizationUniversal (COCD_TP145203GB03), RecipientPersonUniversal (COCD_TP145202GB02) and RecipientWorkgroupUniversal (COCD_TP145204GB03) have been added to this Recipient Constraint Choice.
    • The Patient Constraint Choice (NPFIT-800019#Role) has been upversioned for later domains, since the template, ChildPatientUniversal (COCD_TP145230GB01) have been added to this Patient Constraint Choice.
    • The tabular view description of NullifyDocument (COCD_TP146049UK05) has been updated to include the "Model Attribute Description" and the "Implementation in XML Instance".
  • Vocabulary
    • The vocabularies, GuardianRoleType and GuardianOrganisationType are included as they are used by the ChildPatientUniversal (COCD_TP145230GB01) template.
    • The vocabularies, ManufacturerModelName and SoftwareName are included as they are used by the AuthorDeviceUniversal (COCD_TP145207GB01) template.
  • Schema
    • Corrected the COCD_TP145201GB01.xsd schema to allow the null flavour to be used when the patient's gender cannot be sent.
  • Configuration Profile
    • This has been upversioned to urn:nhs-en:profile:NullifyDocument-v6-0 reflect the addition of templates to the Recipient Constraint Choice (NPFIT-800008#Role) and to the Patient Constraint Choice (NPFIT-800019#Role).

Version: 2.1; Status: Final

  • Specification
    • The status of the ITK Correspondence pack that uses this associated specification domain has been changed to final. This associated specification domain's status has also been changed to final to reflect this change.
